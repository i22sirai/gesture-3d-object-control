# 手のジェスチャーによる3Dオブジェクト操作インターフェース

MediaPipe HandsとUnityを用いた、手のジェスチャーによる3Dオブジェクト操作システムです。

将来的なXR・空間コンピューティング環境への応用を見据え、キーボードやマウスに依存せず、身体の動きによって3Dオブジェクトを直感的に編集できるインターフェースを研究しています。

> 本プロジェクトは、卒業研究として開発中のプロトタイプです。

## 概要

Webカメラから取得した映像を、Python上のMediaPipe Handsで処理し、手のランドマークを取得します。

取得した手の位置や距離の変化から操作情報を生成し、UDP通信によってUnityへ送信します。Unity側では、受信した情報を3Dオブジェクトの拡大縮小や回転にリアルタイムで反映します。

現在は、以下の基本操作を実装しています。

- 両手の距離変化による拡大縮小
- 右手人差し指先の動きによる回転
- PythonとUnity間のUDP通信
- Unity上の3Dオブジェクトへのリアルタイム反映

---

## 研究の背景・目的

一般的な3D編集ソフトでは、マウスやキーボードを用いて、対象や機能を選択しながら操作します。そのため、初めて使用する人には、操作方法やショートカットを覚えるための学習負担が生じます。

本研究では、手のジェスチャーを入力として利用することで、初めて使用する人でも操作内容を理解しやすく、学習負担や誤操作の少ない3D操作インターフェースの実現を目指しています。

特に、次の点を重視しています。

- 身体の動きと3D操作の対応が直感的であること
- 現在行っている操作を利用者が理解しやすいこと
- 複数の操作を追加しても、操作体系が複雑になりすぎないこと
- 手の認識結果の揺れによる誤操作を抑えること

## システム構成

```text
Webカメラ
    │
    ▼
Python / OpenCV
    │
    ▼
MediaPipe Hands
手のランドマーク取得
    │
    ▼
操作情報の生成
    │
    │ UDP通信
    ▼
Unity / C#
    │
    ▼
3Dオブジェクトへの操作反映
```

### Python側

1. Webカメラから映像を取得
2. MediaPipe Handsで手のランドマークを検出
3. 両手の距離や指先の位置変化を計算
4. Unityへ送信する操作情報を生成
5. UDP通信でUnityへ送信

### Unity側

1. Pythonから送信されたデータを受信
2. 受信した値を操作量へ変換
3. 対象オブジェクトのスケールや回転に反映

## 操作方法

### 拡大縮小

両手の距離変化を利用して、3Dオブジェクトの大きさを変更します。

- 両手を広げる：拡大
- 両手を近づける：縮小

両手間の距離そのものではなく、操作開始時からの距離変化を利用することで、手の動きをオブジェクトのスケールへ反映しています。

### 回転

右手人差し指先の位置変化を利用して、3Dオブジェクトを回転させます。

現在は、指先の細かな揺れによって意図しない回転が発生しないよう、回転量や判定条件の調整を進めています。

## 実行方法

### 1. リポジトリを取得する

Gitを使用する場合は、次のコマンドを実行します。

```bash
git clone https://github.com/i22sirai/gesture-3d-object-control.git
cd gesture-3d-object-control
```

Gitを使用しない場合は、GitHubの「Code」からZIPファイルをダウンロードし、展開してください。

### 2. Pythonライブラリをインストールする

```bash
pip install opencv-python mediapipe
```

環境によっては、`pip`の代わりに`pip3`を使用してください。

```bash
pip3 install opencv-python mediapipe
```

### 3. Unityプロジェクトを開く

1. Unity Hubを起動します。
2. リポジトリ内の`Unity`フォルダをプロジェクトとして追加します。
3. 使用するシーンを開きます。
4. Unityの再生ボタンを押します。

### 4. Pythonプログラムを実行する

別のターミナルを開き、`Python`フォルダへ移動します。

```bash
cd Python
```

Pythonプログラムを実行します。

```bash
python main.py
```

環境によっては、次のコマンドを使用してください。

```bash
python3 main.py
```

Webカメラが起動し、手の認識が開始されます。生成された操作情報はUDP通信によってUnityへ送信されます。

### 5. カメラが起動しない場合

使用環境によって、カメラ番号が異なる場合があります。

`Python/main.py`内の次の部分を確認してください。

```python
cv2.VideoCapture(1)
```

内蔵カメラを使用する場合は、次のように`0`へ変更すると起動することがあります。

```python
cv2.VideoCapture(0)
```

## 3層構造について

複数の3D編集操作を、限られたジェスチャーから扱えるようにするため、今後は操作体系に3層構造を導入する予定です。

操作を段階的に整理することで、次の実現を目指します。

- ジェスチャーの種類を増やしすぎずに複数の操作を扱う
- 利用者が現在の操作状態を理解しやすくする
- 異なる操作が同時に認識されることによる誤操作を減らす
- 基本操作と詳細操作を段階的に切り替えられるようにする

各層の具体的な役割や遷移方法については、現在設計を進めています。

## 現在の課題

- 手の認識結果に細かな揺れが発生する
- 拡大縮小量が大きく変化する場合がある
- 回転方向や回転量が安定しない場合がある
- 意図していない操作が認識される場合がある
- 操作数が増えた際の切り替え方法を整理する必要がある
- 初めて使用する人にも理解しやすいUIを設計する必要がある

今後は、しきい値の調整、値の平滑化、操作状態の明示などによって改善を進めます。
